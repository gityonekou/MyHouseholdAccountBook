# MyHouseholdAccountBook(1.0.0-dev 版)
マイ家計簿のWebアプリケーションです。<br>
SpringBootを使って作成しています。<br>
バージョン1.0.0以降、生成AI(Claude Code)を使っています。<br>
<br>
　B版のソースをもとに、生成AIにお手伝いいただきながらリファクタリング（DDD完全適応)しながらバージョン1.0.0を作成中です。（生成AIすごすぎます。。）<br>
　リファクタリングの参考本として「良いコード、悪いコード」と「現場に役立つシステム設計の原則」を読み終わったので、生成AIにて作成してもらったものを<br>
　リファクタリング観点で見て十分問題なさそうならOKとしています。（まあ、自分が指摘した内容はまだまだ少ないのが現状ではあります）<br>
　テストはJUnitで単体・結合まで実施します。（テストケースのソースも作ってもらってます）<br>
　<br>
　個人で使ってていろいろ追加したい機能、直したいところが出てきたので、DDD完全順守後にいろいろ追加します。<br>
　DDD順守したソースにしてよかったと実感できるようになりたいですね。<br>
　＜追加予定の機能＞<br>
　・買い物登録詳細機能の作成など、B版でメニューだけ作成して、機能の取込が出来ていないものの機能取込<br>
　（現状、簡易版の買い物登録で運用中だったりします。。。）<br>
　・イベントに応じた買い物を登録できるようにする<br>
　・買い物登録の残は携帯(アンドロイド側）のアプリもkotlinで作るなど、、<br>
<br>
