## 1. システム概要

### 1.1 システムの目的

マイ家計簿は、個人の収支を管理し、家計の見える化を実現するWebアプリケーションです。日々の支出・収入を記録し、月次・年次での収支分析を可能にします。

### 1.2 システムの特徴

- **個人向け家計管理**: 個人ユーザーごとの収支管理
- **階層的な支出分類**: 最大5階層の支出項目による詳細な分類
- **買い物登録機能**: 日常の買い物を簡単に記録
- **固定費管理**: 定期的な支出の管理
- **イベント管理**: 特別なイベントに関連する支出の管理
- **月次・年次集計**: 期間ごとの収支分析

### 1.3 想定ユーザー

- **主要ユーザー**: 個人（家計を管理したい人）
- **ユーザー数規模**: 個人利用想定（マルチユーザー対応）
- **利用環境**: Webブラウザ（PC・タブレット・スマートフォン）

## 2. 機能要件

### 2.1 認証・ユーザー管理

#### FR-AUTH-001: ログイン機能
- **概要**: ユーザーID・パスワードによるログイン認証
- **入力**:
  - ユーザーID
  - パスワード
- **処理**:
  - Spring Securityによる認証
  - セッション管理
- **出力**:
  - 認証成功時: トップメニュー画面へ遷移
  - 認証失敗時: エラーメッセージ表示

#### FR-AUTH-002: ログアウト機能
- **概要**: セッションの破棄とログアウト
- **処理**: セッション無効化
- **出力**: ログイン画面へ遷移

### 2.2 収支照会機能

#### FR-INQ-001: 月次収支照会
- **概要**: 指定月の収支情報を表示
- **機能ID**: AccountMonthInquiryController
- **入力**:
  - 対象年月（YYYYMM形式）
- **表示項目**:
  - 収入金額
  - 積立金取崩金額
  - 支出金額
  - 支出予定金額
  - 収支金額（収入 - 支出）
  - 支出項目別の金額一覧（階層表示）
    - 支出項目名
    - 支出金額
    - 支出金額B（特定条件）
    - 支出金額C（特定条件）
    - 支出金額B+C
    - 各項目の構成比率
    - 支払日
- **操作**:
  - 前月・次月への移動
  - 買い物登録画面への遷移
  - 収支登録画面への遷移

#### FR-INQ-002: 年次収支照会
- **概要**: 年間の収支サマリーを表示
- **機能ID**: AccountYearInquiryController
- **入力**:
  - 対象年（YYYY形式）
- **表示項目**:
  - 月別の収支金額
  - 年間合計
  - 月別グラフ（オプション）
- **操作**:
  - 前年・次年への移動
  - 月次詳細への遷移

#### FR-INQ-003: 年次明細照会
- **概要**: 年間の支出明細を一覧表示
- **入力**:
  - 対象年
  - 支出項目（フィルター）
- **表示項目**:
  - 支出明細一覧
  - 月別集計

#### FR-INQ-004: 支払確認
- **概要**: 未払い・支払済みの確認
- **機能ID**: PaymentConfirmationController
- **表示項目**:
  - 支払予定日
  - 支出項目
  - 金額
  - 支払状態

### 2.3 収支登録機能

#### FR-REG-001: 収入・支出登録
- **概要**: 月次の収入・支出を一括登録
- **機能ID**: IncomeAndExpenditureRegistController
- **入力**:
  - 対象年月
  - 収入情報
    - 収入区分
    - 収入名称
    - 収入金額
    - 取得日
  - 支出情報
    - 支出項目
    - 支出名称
    - 支出金額
    - 支払日
    - イベント（オプション）
- **処理**:
  - 入力値バリデーション
  - 収支テーブルへの登録/更新
  - 収入テーブルへの登録
  - 支出テーブルへの登録
  - 支出金額テーブルの集計更新
- **バリデーション**:
  - 金額の妥当性チェック
  - 日付の妥当性チェック
  - 必須項目チェック

#### FR-REG-002: 収支更新
- **概要**: 既存の収支データの更新
- **入力**: 更新対象データ
- **処理**: トランザクション内での一括更新

## 5. データ要件

### 5.1 データ保持期間

| データ種別 | 保持期間 | 備考 |
|-----------|---------|------|
| 収支データ | 無期限 | ユーザーが削除するまで |
| マスタデータ | 無期限 | 論理削除対応 |
| セッション | セッション終了まで | - |

### 5.2 データバックアップ

- **頻度**: ユーザー責任（推奨: 月次）
- **方法**: データベースエクスポート機能（別途実装）

### 5.3 データ整合性

#### データ整合性ルール

1. **収支金額の一致**
   ```
   収入金額 + 積立金取崩金額 - 支出金額 = 収支金額
   ```

2. **収入合計の一致**
   ```
   収支テーブル.収入金額 = SUM(収入テーブル.収入金額)
   ```

3. **支出合計の一致**
   ```
   収支テーブル.支出金額 = SUM(支出テーブル.支出金額)
   ```

4. **支出項目別集計の一致**
   ```
   SUM(支出金額テーブル.支出金額) = SUM(支出テーブル.支出金額)
   ```

### 5.4 マスタデータ初期値

- **支出項目BASE**: システム標準の支出項目分類
- **店舗BASE**: 主要な店舗情報
- **新規ユーザー登録時**: BASEテーブルからユーザーテーブルへコピー

## 6. セキュリティ要件

### 6.1 認証・認可

#### NFR-SEC-001: 認証方式
- **方式**: フォーム認証（Spring Security）
- **パスワード**: ハッシュ化して保存

- **アクセス制御**: ユーザーは自分のデータのみアクセス可能

- **対象データ**: ユーザー情報、収支データ

#### NFR-SEC-004: 通信の暗号化
- **推奨**: HTTPS通信
- **本番環境**: SSL/TLS証明書の使用を推奨

### 6.3 入力値検証

#### NFR-SEC-005: バリデーション
- **サーバーサイド**: 必須実装
- **クライアントサイド**: UX向上のため推奨

## 7. 制約事項

### 7.1 技術的制約

- **Java**: バージョン17以上
- **Spring Boot**: 3.0.7
- **データベース**: MariaDB（本番）、H2（テスト）
- **ブラウザ**: モダンブラウザ（Chrome、Firefox、Safari、Edge）
- **ORM**: MyBatis（JPAは使用しない）

### 7.2 機能的制約

- **マルチユーザー**: 対応済みだが、基本は個人利用想定
- **多通貨**: 非対応（日本円のみ）
- **国際化**: 非対応（日本語のみ）
- **モバイルアプリ**: 非対応（Webブラウザのみ）

### 7.3 運用上の制約

- **ホスティング**: ユーザー責任
- **データバックアップ**: ユーザー責任
- **アップデート**: 手動実施
